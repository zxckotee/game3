## Версия 0.9.7 (18.04.2025)
### Исправлено:
- Синхронизированы имена торговцев для корректной работы системы скидок
  - "Старый Чен" переименован в "Торговец Чен"
  - "Старейшина Ву" переименован в "Старейшина Чжан"
- Теперь скидки у торговцев правильно применяются на основе социальных отношений

## Версия 0.9.6 (18.04.2025)
### Изменено:
- Упрощена система скидок на рынке: теперь скидка рассчитывается только по формуле лояльность/10
- Убраны наценки для игроков с низкой лояльностью
- Добавлена функция integrateRelationshipSyncer для стабильной работы синхронизации отношений

## Версия 0.9.5 (17.04.2025)
### Улучшено:
- Смягчена система наценок для игроков с низкой лояльностью (штраф уменьшен в 4 раза):
  - Низкая лояльность (≤20%): наценка к ценам по формуле (20 - лояльность)/4%
  - Средняя лояльность (21-50%): стандартные цены (без изменений)
  - Высокая лояльность (>50%): скидка по формуле (лояльность/10)% (без изменений)

## Версия 0.9.4 (17.04.2025)
### Добавлено:
- Новая трехуровневая система скидок и наценок на рынке в зависимости от лояльности:
  - Низкая лояльность (≤20%): наценка к ценам по формуле (20 - лояльность)%
  - Средняя лояльность (21-50%): стандартные цены
  - Высокая лояльность (>50%): скидка по формуле (лояльность/10)%
- Двусторонняя синхронизация между лояльностью секты и социальными отношениями
- Обновление лояльности членов секты синхронизируется с социальными отношениями
- Обновление социальных отношений синхронизируется с лояльностью в секте

### Улучшения:
- Более наглядное отображение скидок и наценок в интерфейсе магазина
- Визуальное отличие положительных скидок и отрицательных наценок
- Повышение производительности синхронизации отношений

### Исправлено:
- Исправлены ошибки, которые могли возникать при обновлении лояльности секты
- Исправлена проблема с отображением некорректных цен у торговцев
